# 产品范围、用户与交付边界

## 背景与目标

综合研发智能平台面向团队研发协作、架构治理和 AI 辅助开发。平台不是单纯的代码图谱工具，也不是单纯的知识库工具，而是把代码事实、团队知识、可信检索、上下文包和变更影响分析串成闭环。

平台核心目标：

- 帮助团队理解项目代码、接口、模块和调用关系。
- 沉淀可确认、可复审、可追溯的研发知识资产。
- 统一检索代码、文档、知识资产和接口证据。
- 为 Codex、Cursor、Claude Code 等 AI 编程助手生成可信上下文包。
- 基于代码变更初步判断影响范围、风险点和测试建议。

## 产品要回答的问题

1. 项目结构、接口、调用链、数据访问事实是什么。
2. 团队确认过哪些业务规则、架构决策、故障经验和开发规范。
3. 当前任务需要哪些可信知识、代码证据和接口上下文。
4. 一次代码变更可能影响哪些接口、模块、知识资产和测试范围。
5. 团队知识是否过期、缺失、低质量或需要复审。

## 设计原则

- 以项目空间为基础隔离数据、权限、索引、任务和知识资产。
- 以研发任务为高频入口，串联检索、上下文包和影响分析。
- 代码分析只提供事实和候选证据，关键知识仍需要人工确认。
- 所有智能推荐必须保留来源、证据和解释，避免黑盒结论。
- 先跑通端到端核心闭环，再增强能力深度。
- 当前实现可以复用，但新功能边界不受当前菜单结构限制。

## 目标用户与角色

### 系统管理员

负责平台初始化、用户管理、项目接入、权限分配、系统配置、任务失败排查和审计查看。

### 项目负责人

负责项目空间内成员授权、知识资产确认、复审安排、接口说明、业务规则和 ADR 治理。

### 研发成员

负责日常需求开发、缺陷修复、代码理解、知识补充、可信检索、AI 上下文包生成和变更影响查看。

### 测试/运维成员

关注接口、业务规则、故障案例、变更影响、测试建议、风险提示和排障知识。

### 架构治理角色

负责 ADR、开发规范、模块边界、跨项目经验、技术债、知识质量和复审机制。核心可用版中该职责由项目负责人承担，增强版可扩展为独立权限标签。

## 核心可用版角色收敛

核心可用版采用项目级角色模型：

- 系统管理员
- 项目负责人
- 项目成员
- 只读成员

研发成员默认映射为项目成员；测试/运维成员可映射为项目成员或只读成员；架构治理角色在核心可用版中由项目负责人承担。

## 一级功能域

1. 平台底座：账号、角色、权限、项目空间、系统配置、任务、审计。
2. 代码智能底座：代码仓库接入、代码解析、接口识别、调用关系、SQL 线索、Neo4j 图关系。
3. 知识资产中心：知识文档、结构化知识资产、状态流转、证据绑定、确认、复审。
4. 统一检索与证据中心：Qdrant 索引、可信检索、证据链、SearchLog。
5. AI 研发助手工作台：任务输入、上下文召回、上下文包、导出审计。
6. 变更影响与治理运营：diff 影响初判、ReviewTask、治理看板、审计。

## 核心闭环必备能力

- 多用户登录、角色权限、项目授权。
- Git 项目接入、更新、项目空间隔离。
- Java/Spring/MyBatis/前端 API 的基础代码事实提取。
- Neo4j 项目内基础图关系写入和基础查询。
- 知识文档和结构化知识资产管理。
- 知识确认、待复审、归档等基础状态流转。
- 代码、知识文档、知识资产的 Qdrant 统一索引。
- 可信检索，结果区分来源、状态和证据。
- AI 上下文包，用户输入任务后生成可复制上下文。
- 变更影响初判，基于文件列表或 diff 输出影响接口、知识资产和测试建议。
- 基础任务状态和失败原因。

## 增强能力

- 复杂知识评审流、复审提醒和知识版本历史。
- 知识质量评分、重复知识、冲突知识和标签治理。
- 组织级看板、跨项目检索和组织级知识地图。
- PR/MR 集成、webhook、CI 自动触发和风险报告。
- 多语言、多框架深度分析。
- 独立 InterfaceAsset、参数级接口治理、错误码治理。
- 交互式图谱页面、复杂路径探索和子图导出。
- 内置 AI 问答、多轮会话、MCP/API 对接。

## 当前产品边界

当前产品边界不要求：

- 完整图数据库平台或组织级知识图谱治理系统。
- 全语言、全框架深度静态分析。
- 自动修改代码或自动提交代码。
- 复杂多级审批流。
- 企业多租户计费。
- 自动判断知识真假。
- 把影响分析结果包装成绝对确定结论。

## 核心首轮价值目标

首轮应优先验证“可信上下文包是否有用”：

```text
项目接入
  -> 代码事实
  -> 知识证据
  -> Qdrant + Neo4j 可追溯查询
  -> AI 上下文包
  -> 导出审计
```

变更影响和治理功能应作为后续核心切片接入。
