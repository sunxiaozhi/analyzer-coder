# 平台、项目与权限需求

## 功能目标

平台底座负责账号、角色、权限、项目空间、项目配置、系统配置、任务状态和审计。所有业务数据必须以项目空间为边界隔离。

## 功能点

| 功能域 | 功能点 |
|---|---|
| 账号体系 | 登录、退出、当前用户、密码修改、账号启停、管理员创建账号 |
| 角色体系 | 系统管理员、项目负责人、项目成员、只读成员 |
| 权限控制 | 项目访问权限、模块访问权限、操作权限、管理员权限校验 |
| 项目空间 | 项目创建、项目切换、项目配置、项目数据隔离、项目归档 |
| 数据源接入 | Git 仓库、默认代码路径、知识文档目录 |
| 系统配置 | MySQL、Qdrant、Neo4j、embedding、对象存储、路径策略 |
| 后台任务 | 分析任务、索引任务、上下文生成任务、影响分析任务、任务状态、失败重试 |
| 审计日志 | 登录、权限变更、项目操作、知识确认、任务执行、上下文导出 |

## 核心对象

### Project

项目空间。关键字段：

- `id`
- `projectKey`
- `name`
- `gitUrl`
- `branch`
- `workspacePath`
- `defaultCodePath`
- `defaultKnowledgePath`
- `ownerUserId`
- `status`
- `createdAt`
- `updatedAt`

### ProjectMember

用户在项目内的角色。关键字段：

- `projectId`
- `userId`
- `role`
- `status`
- `createdAt`
- `updatedAt`

### RepositorySnapshot

仓库快照。记录一次同步或分析基线。关键字段：

- `id`
- `projectId`
- `branch`
- `commitHash`
- `commitMessage`
- `syncedAt`
- `analyzedAt`
- `sourcePaths`
- `status`

### AuditEvent

审计事件。关键字段：

- `id`
- `projectId`
- `actorUserId`
- `action`
- `targetType`
- `targetId`
- `detail`
- `createdAt`
- `ipAddress`

## 项目接入闭环

```text
系统管理员创建项目
  -> 填写 Git 地址、分支、默认代码路径、默认知识库路径
  -> 指定项目负责人和成员
  -> 校验项目名、Git 地址、分支和路径策略
  -> 建立 .analzer_projects 项目工作区
  -> clone 或 pull 仓库
  -> 创建 RepositorySnapshot
  -> 写入 Project / ProjectMember / AuditEvent
```

Git 拉取失败时，项目进入 `sync_failed` 状态，失败原因保留，用户可重试同步。

## 权限规则

- 所有项目级对象必须包含 `projectId`。
- 后端必须校验用户是否有项目访问权限。
- 前端隐藏按钮只作为体验优化，不能作为权限依据。
- 普通用户不能查看、检索、导出或分析未授权项目。
- 系统管理员可以进入任意项目排查问题，但必须记录审计。
- 项目负责人管理本项目成员、路径、知识和治理配置。
- 项目成员可触发分析、索引、检索、上下文包和影响初判。
- 只读成员默认不能导出上下文包、触发分析、索引、影响分析，也不能创建或确认知识。

## 页面与操作

| 页面 | 主要操作 | 关键展示 | 状态要求 |
|---|---|---|---|
| 登录页 | 用户名密码登录 | 失败原因、登录后项目入口 | 登录失败、无权限 |
| 项目列表 | 创建项目、选择项目、归档项目 | 项目列表、分支、负责人、状态 | 空项目、同步失败 |
| 项目配置 | 修改默认路径、查看仓库与分支、管理成员 | 路径配置、成员角色、最近快照 | 路径非法、无权限 |
| 账号权限 | 创建用户、启停账号、授权项目 | 用户、角色、项目权限 | 管理员校验 |
| 系统配置 | 配置 MySQL、Qdrant、Neo4j、embedding、对象存储、路径策略 | 连接状态、模型状态、存储状态 | 配置缺失、连接失败 |

## 验收标准

- 系统管理员可以创建项目并指定负责人。
- 用户只能看到被授权项目。
- 项目负责人可以管理本项目成员。
- 未授权用户访问项目 API 返回 403。
- 系统管理员可以查看、保存并测试 MySQL、Qdrant、Neo4j、embedding、对象存储和路径策略配置。
- 项目同步失败时，已有项目数据不被清空，失败原因可查看并可重试。
