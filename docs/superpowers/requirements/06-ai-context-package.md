# AI 上下文包需求

## 功能目标

AI 研发助手工作台负责把研发任务、可信知识、代码事实、接口、SQL、调用链、风险点和测试建议组织成可复制的 Markdown 或纯文本上下文包。核心可用版只生成上下文包，不直接调用大模型写代码。

## 功能点

| 功能域 | 功能点 |
|---|---|
| 任务入口 | 需求开发、缺陷修复、重构、接口变更、排障、代码评审 |
| 任务理解 | 用户输入任务目标、模块、约束、相关文件、期望输出 |
| 上下文召回 | 检索相关业务规则、代码、接口、调用链、规范、故障案例 |
| 上下文选择 | 自动推荐、人工勾选、排除低可信内容、控制长度 |
| 上下文包生成 | 面向 Codex、Cursor、Claude Code 的复制格式 |
| 风险与测试 | 风险点、测试建议、待确认问题 |
| 追溯记录 | 引用了哪些知识、代码、接口和版本 |
| 导出审计 | 复制和导出记录审计，导出前权限复检和基础敏感信息过滤 |

## 核心对象

### ContextPackage

关键字段：

- `id`
- `projectId`
- `taskType`
- `taskText`
- `selectedItems`
- `excludedItems`
- `content`
- `format`
- `createdBy`
- `createdAt`
- `exportedAt`

## 上下文包闭环

```text
输入任务
  -> 根据任务类型生成召回策略
  -> 调用可信检索生成候选上下文
  -> 记录 SearchLog 和候选来源
  -> 用户勾选、排除、排序
  -> 生成上下文包正文
  -> 保存 ContextPackage 和引用清单
  -> 写入 Neo4j INCLUDES 关系
  -> 导出前执行权限复检和基础敏感信息过滤
  -> 导出时写入 AuditEvent
```

## 上下文包内容

上下文包必须包含：

- 任务描述和用户约束。
- 相关已确认知识资产。
- 相关待复审知识及风险标识。
- 相关代码片段、接口、调用链和 SQL 线索。
- 相关开发规范、故障案例和测试建议。
- 来源追溯和生成时间。

## 任务召回策略

- 需求开发：优先召回业务规则、接口、模块说明、开发规范。
- 缺陷修复：优先召回故障案例、相关代码、历史知识和 SQL 线索。
- 接口变更：优先召回前后端映射、接口说明、测试建议。
- 重构：优先召回模块边界、调用关系、ADR、规范。
- 排障：优先召回故障案例、日志线索、接口和 SQL。
- 代码评审：优先召回规范、风险点、变更影响和相关知识。

## 边界

- 上下文包用于组织可信上下文，不负责自动写代码。
- 基础能力支持 Markdown 或纯文本复制格式。
- Codex、Cursor、Claude Code 专用格式、API 或 MCP 属于增强能力。
- 企业级敏感信息治理属于增强能力；核心版只做基础敏感信息过滤。

## 页面与操作

| 页面 | 主要操作 | 关键展示 |
|---|---|---|
| AI 研发助手 | 输入任务、召回候选、勾选排除排序、生成上下文包、复制、导出 | 任务描述、已选上下文、风险提示、引用来源 |

## 验收标准

- 用户可以输入任务描述并召回候选上下文。
- 用户可以勾选、排除、排序上下文项。
- 系统生成 Markdown 或纯文本上下文包。
- 上下文包保存引用清单，不能只保存最终文本。
- 上下文包生成后，系统写入 ContextPackage 与引用对象的 Neo4j 关系。
- 导出上下文包记录审计。
- 导出前执行权限复检和基础敏感信息过滤。
