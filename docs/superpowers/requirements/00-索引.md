# 综合研发智能平台需求索引

本目录是综合研发智能平台需求的唯一维护入口。需求按功能域拆分，不再保留单独的总需求文档。

需求文档回答“要做什么、为什么做、验收什么”；对应详细设计文档回答“怎么设计、对象怎么流转、接口和状态如何落地”。

## 推荐阅读路径

### 了解全貌

```text
01-产品范围与角色
  -> 10-端到端功能闭环
  -> 09-架构非功能与验收
```

### 评审 MVP

```text
10-端到端功能闭环
  -> 01-产品范围与角色
  -> 02-平台项目与权限
  -> 03-代码智能与图关系
  -> 04-知识资产与证据
  -> 05-统一索引与可信检索
  -> 06-AI上下文包
  -> 09-架构非功能与验收
```

首轮 MVP 聚焦“项目内可信上下文闭环”，不强依赖完整变更影响、ReviewTask 和治理看板。

### 评审单个功能

```text
先看本目录中的需求文档
  -> 再看 ../designs/ 中同编号设计文档
  -> 如涉及跨模块对象流转，再看 10-端到端功能闭环 和 ../designs/10-端到端对象流转
  -> 如涉及安全、状态、验收或非功能，再看 09-架构非功能与验收
```

## 需求分层

| 层级 | 文档 | 作用 |
|---|---|---|
| 产品层 | `01` | 定义平台边界、角色、核心版和增强版范围 |
| 功能层 | `02` 至 `08` | 定义每个业务功能域的目标、功能点、核心对象、页面操作和验收标准 |
| 横向链路层 | `10` | 定义功能之间如何串联，避免单点功能无法闭环 |
| 架构验收层 | `09` | 定义所有功能共同遵守的架构、非功能、业务规则和验收用例 |

## 端到端主链路

```text
项目接入
  -> 仓库同步
  -> 代码分析
  -> CodeFact 入库
  -> Neo4j 基础图关系
  -> KnowledgeAsset 和 Evidence
  -> Qdrant 统一索引
  -> 可信检索
  -> AI 上下文包
  -> 变更影响分析
  -> ReviewTask 或治理项
  -> 知识/Evidence 更新
  -> 重新索引和更新图关系
```

## MVP 边界

首轮 MVP 要求跑通：

```text
项目和权限
  -> 代码事实和 Neo4j 基础关系
  -> 知识资产和 Evidence
  -> Qdrant 可信检索
  -> AI 上下文包
  -> 导出审计
```

第二个核心切片接入：

- 变更影响分析。
- ReviewTask。
- 治理看板。
- 低命中治理。

## 维护规则

- 新增或修改功能需求时，必须同步检查同编号设计文档。
- 修改跨模块链路、对象输入输出或 MVP 边界时，必须同步更新 `10-端到端功能闭环.md`。
- 修改安全、权限、状态、任务、非功能或验收口径时，必须同步更新 `09-架构非功能与验收.md`。
- 不再新增总需求文档；全局内容应归入 `01`、`09` 或 `10`。
- 需求文档避免描述具体接口实现，接口、服务、状态机和对象字段细节放到 `../designs/`。
