# 综合研发智能平台需求拆分索引

## 核心首轮建议

首轮只验证“项目内可信上下文闭环”：

```text
项目接入
  -> 代码事实和 Neo4j 基础图关系
  -> 知识资产和 Evidence
  -> Qdrant 可信检索
  -> AI 上下文包
  -> 导出审计
```

变更影响、ReviewTask、治理看板作为后续核心切片接入，不应阻塞首轮上下文包价值验证。

完整端到端链路见 `10-端到端功能闭环.md`。
