# 代码智能页面原型

## 代码事实

### 页面目标

查看项目代码分析结果，触发分析，定位代码事实，绑定证据或加入上下文。

### 线框

```text
页面标题：代码事实                         [触发分析] [导出图数据]
状态条：快照 abc123 | 分析状态 partial_failed | 失败文件 12 | Neo4j ready

筛选区：
  类型 [全部] 文件路径 [____] 符号 [____] 置信度 [全部]

┌────────┬────────────┬──────────────┬────────────┬────────┬────────┐
│ 类型   │ 文件        │ 符号          │ 行区间      │ 置信度 │ 操作   │
├────────┼────────────┼──────────────┼────────────┼────────┼────────┤
│ endpoint│ UserCtl.java│ GET /users   │ 22-38      │ 高     │ 详情   │
│ sql    │ UserMapper  │ selectById    │ 11-20      │ 中     │ 详情   │
└────────┴────────────┴──────────────┴────────────┴────────┴────────┘
```

### 主要操作

- 触发代码分析。
- 查看失败文件。
- 按类型、路径、符号筛选。
- 打开代码事实详情。
- 绑定为知识证据。
- 加入上下文包候选。
- 查看基础图关系。
- 导出裁剪后的 JSON 图数据。

### 状态

| 状态 | 表现 |
|---|---|
| 未分析 | 空状态提示先触发分析 |
| 分析中 | 状态条展示运行中，不清空旧结果 |
| 部分失败 | 展示成功结果，提供失败文件入口 |
| Neo4j 写入失败 | 图关系入口降级，显示重试图写入 |

## 代码事实详情抽屉

### 线框

```text
抽屉：代码事实详情

基础信息
  类型、文件、符号、签名、行区间、快照、置信度

代码片段
  只读代码块，突出行区间

关联关系
  调用方 / 被调用方 / Endpoint / SQL / Evidence / KnowledgeAsset

操作
  [绑定证据] [加入上下文包] [查看图关系] [复制引用]
```

### 规则

- 详情内所有关联对象都必须可跳转。
- 绑定证据时只能绑定当前项目知识资产。
- 加入上下文包时传 `sourceType`、`sourceId`、`chunkId`，不只传文本。

## 接口关系

### 页面目标

展示 Spring endpoint、前端调用、匹配关系、接口风险和接口说明入口。

### 线框

```text
页面标题：接口关系
筛选：HTTP 方法 [全部] 路径 [____] 匹配状态 [全部] 风险 [全部]

┌────────┬────────────────┬────────────┬────────────┬────────┬────────┐
│ 方法   │ 路径            │ 后端方法    │ 前端调用数   │ 风险   │ 操作   │
├────────┼────────────────┼────────────┼────────────┼────────┼────────┤
│ GET    │ /api/users/{id} │ getUser     │ 3          │ 低     │ 详情   │
│ POST   │ /api/orders     │ createOrder │ 0          │ 缺调用 │ 详情   │
└────────┴────────────────┴────────────┴────────────┴────────┴────────┘
```

### 详情区域

```text
接口详情
  Endpoint 信息
  前端调用列表
  SQL 线索
  关联知识资产 api_doc
  Evidence
  测试建议入口
```

### 主要操作

- 查看 endpoint 详情。
- 查看前端调用。
- 创建或关联 `api_doc` 知识资产。
- 加入上下文包。
- 查看图关系。

## 基础图关系抽屉

### 页面目标

展示当前对象的有限深度邻居和路径，不建设复杂图谱工作台。

### 线框

```text
抽屉：基础图关系

中心对象：GET /api/users/{id}
深度：[1] [2]
关系类型：[CALLS] [USES_SQL] [EVIDENCES] [SUPPORTS]

左侧：节点列表
右侧：关系路径表
  起点 -> 关系 -> 终点 -> 置信度 -> 原因

[导出 JSON]
```

### 限制

- 必须按 `projectId` 查询。
- 默认深度 1，最大深度由后端限制。
- 节点数量超限时提示已裁剪。
