# 组织层页面原型

## 登录页

### 页面目标

完成用户名密码登录，并把用户引导到最近访问项目或项目列表。

### 线框

```text
┌──────────────────────────────┐
│ 综合研发智能平台              │
│                              │
│ 用户名 [________________]     │
│ 密码   [________________]     │
│                              │
│ [ 登录 ]                      │
│                              │
│ 错误提示 / 账号停用提示       │
└──────────────────────────────┘
```

### 状态

| 状态 | 表现 |
|---|---|
| 登录失败 | 表单下方显示失败原因 |
| 账号停用 | 明确提示联系管理员 |
| 已登录访问 | 自动进入最近项目或项目列表 |

## 项目列表

### 页面目标

展示当前用户可访问项目，支持创建、筛选、进入和归档项目。

### 线框

```text
页面标题：项目列表                         [新建项目]
筛选区：项目名 [____] 状态 [全部] 负责人 [全部]

┌──────────────┬────────┬────────┬────────┬─────────────┬────────┐
│ 项目名称      │ 分支   │ 状态   │ 负责人 │ 最近快照      │ 操作   │
├──────────────┼────────┼────────┼────────┼─────────────┼────────┤
│ order-service│ main   │ active │ 张三   │ abc123       │ 进入   │
│ legacy-crm   │ dev    │ failed │ 李四   │ 同步失败      │ 重试   │
└──────────────┴────────┴────────┴────────┴─────────────┴────────┘
```

### 主要操作

- 进入项目。
- 新建项目。
- 按项目名、状态、负责人筛选。
- 查看同步失败原因。
- 管理员归档或恢复项目。

### 状态

| 状态 | 表现 |
|---|---|
| 无项目 | 显示空状态；管理员显示新建项目按钮 |
| 同步失败 | 行内红色状态，操作区显示重试 |
| 已归档 | 行内灰色标签，默认隐藏，可通过状态筛选查看 |

## 新建项目弹窗

### 页面目标

创建项目、配置仓库和初始成员。

### 线框

```text
弹窗：新建项目

基础信息
  项目名称 [____________]
  描述     [____________]

代码仓库
  Git 地址 [____________]
  分支     [main________]
  默认代码路径 [./_______]
  默认知识库路径 [docs____]

成员
  项目负责人 [选择用户]
  初始成员   [选择用户，多选]

底部：[取消] [测试连接] [创建项目]
```

### 校验

- 项目名必填且不可重复。
- Git 地址和分支必填。
- 路径必须通过 workspace/project root 校验。
- 创建前可以测试 Git 连接。

## 账号与权限

### 页面目标

系统管理员维护用户、启停账号、重置密码、授权项目和设置项目角色。

### 线框

```text
页面标题：账号与权限                       [新建用户]
筛选：用户名 [____] 状态 [全部] 项目 [全部]

左侧：用户列表
┌────────┬────────┬────────┬────────┐
│ 用户名 │ 状态   │ 系统角色│ 操作   │
└────────┴────────┴────────┴────────┘

右侧：用户权限详情
  基础信息
  项目授权表
  ┌──────────────┬────────────┬────────┐
  │ 项目          │ 项目角色    │ 操作   │
  └──────────────┴────────────┴────────┘
```

### 主要操作

- 新建用户。
- 启用或停用用户。
- 重置密码。
- 设置系统管理员。
- 授权项目。
- 修改项目内角色：负责人、成员、只读。

### 状态

| 状态 | 表现 |
|---|---|
| 用户停用 | 不允许登录，权限详情只读 |
| 无项目授权 | 明确显示用户不能访问任何项目 |
| 角色冲突 | 提示一个项目至少保留一个项目负责人 |

## 系统配置

### 页面目标

配置工作区、状态存储、Qdrant、Neo4j、embedding、对象存储和路径策略，并查看健康状态。

### 线框

```text
页面标题：系统配置

Tabs: [基础路径] [状态存储] [向量库] [图数据库] [Embedding] [对象存储] [健康检查]

配置表单区
  字段...
  [测试连接] [保存]

右侧健康摘要
  MySQL: 正常
  Qdrant: 正常
  Neo4j: 异常
  Embedding: 未配置
```

### 配置项

| Tab | 内容 |
|---|---|
| 基础路径 | workspace root、项目工作区、路径白名单策略 |
| 状态存储 | MySQL 连接信息或 JSON 状态存储策略 |
| 向量库 | Qdrant 地址、collection、超时、连接测试 |
| 图数据库 | Neo4j 地址、库名、账号、连接测试 |
| Embedding | 服务地址、模型名、维度、连接测试 |
| 对象存储 | 导出文件、报告、上下文包快照存储配置 |
| 健康检查 | 各服务状态、最近失败原因、刷新时间 |

### 安全要求

- 密码、token、私钥不明文回显。
- 保存前执行最小格式校验。
- 连接测试失败不覆盖已有可用配置。
