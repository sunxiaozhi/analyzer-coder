# 变更影响页面原型

## 变更影响

### 页面目标

输入文件列表或 diff，输出候选影响、风险点、测试建议和待复审候选。

### 线框

```text
页面标题：变更影响

输入区
  输入类型 [文件列表 | diff]
  文本框：
    src/main/java/.../OrderService.java
    src/main/resources/mapper/OrderMapper.xml
  [开始分析]

历史任务
┌────────────┬────────────┬────────┬────────────┬────────┐
│ 创建时间    │ 输入类型     │ 状态   │ 置信度      │ 操作   │
└────────────┴────────────┴────────┴────────────┴────────┘
```

### 校验

- 文件列表不能为空。
- diff 格式非法时不创建成功任务。
- 未执行代码分析时提示先分析代码。

## 影响分析详情

### 页面目标

分层展示确定事实、高置信候选、中低置信候选和不确定性说明。

### 线框

```text
页面标题：影响分析详情
状态条：任务 succeeded | 总置信度 中 | 创建人 张三 | 快照 abc123

Tabs:
  [变更文件] [命中代码事实] [影响接口] [影响知识] [风险点] [测试建议] [图路径]

变更文件
┌────────────┬────────────┬────────┬────────┐
│ 文件路径    │ 变更行区间   │ 命中事实│ 操作   │
└────────────┴────────────┴────────┴────────┘

影响知识
┌────────────┬────────┬────────────┬────────────┬────────┐
│ 知识标题    │ 状态   │ 置信度      │ 推荐原因     │ 操作   │
└────────────┴────────┴────────────┴────────────┴────────┘
```

### 分层展示

| 层级 | 展示方式 |
|---|---|
| 确定事实 | 变更文件、变更行、直接命中 CodeFact |
| 高置信候选 | 直接命中 endpoint、SQL、绑定知识 |
| 中置信候选 | Neo4j 一跳或有限深度路径 |
| 低置信候选 | Qdrant 语义召回或目录级推断 |

### 主要操作

- 查看代码事实。
- 查看影响接口。
- 查看知识资产。
- 生成 ReviewTask 候选。
- 导出分析摘要。
- 基于结果创建上下文包。

## 待复审候选

### 页面目标

将命中的已确认知识提交给项目负责人确认是否生成 ReviewTask 或标记待复审。

### 线框

```text
抽屉：待复审候选

触发来源：ChangeAnalysisTask #123
关联知识：订单取消规则
命中原因：OrderService.cancel 发生变更，Evidence 指向该方法
图路径：ChangeTask -> CodeFact -> Evidence -> KnowledgeAsset

操作：
  [生成 ReviewTask]
  [确认无需处理]
  [打开知识详情]
```

### 规则

- 系统只生成候选。
- 最终标记待复审必须由项目负责人确认。
- 忽略候选必须填写原因并写审计。

## 历史记录

### 页面目标

按时间查看影响分析任务。

### 列表字段

- 创建时间。
- 输入类型。
- 变更文件数。
- 命中代码事实数。
- 影响接口数。
- 影响知识数。
- 总置信度。
- 状态。
- 操作。
