# 架构、非功能与验收详细设计

## 总体架构

```text
Browser
  -> Web frontend
  -> Flask API service
  -> MySQL
  -> Qdrant
  -> Neo4j
  -> object storage or file service
  -> .analzer_projects project workspace
```

## 存储职责

| 存储 | 职责 |
|---|---|
| MySQL | 用户、项目、授权、知识资产、模板、任务、审计、上下文包记录和分析记录 |
| Qdrant | 统一语义索引、metadata 过滤、相似度检索和运维 |
| Neo4j | 项目内 CodeFact、接口、SQL、Evidence、KnowledgeAsset、ContextPackage 和 ChangeTask 的基础关系 |
| 对象存储或文件服务 | 分析报告、导出文件、上下文包快照和大体积分析产物 |
| `.analzer_projects` | 项目仓库 clone/pull、分析工作目录和可再生成产物 |

MySQL 保存事实状态，Qdrant 负责语义召回，Neo4j 负责关系追踪和影响路径。三者通过 `projectId`、`snapshotId`、`sourceId` 和 `targetId` 对齐。

## 前端技术设计

| 方向 | 选型 | 用途 |
|---|---|---|
| UI 框架 | Vue 3 | 构建单页 Web 控制台 |
| 构建工具 | Vite | 本地开发、构建和代理 API |
| 组件库 | Element Plus | 表单、表格、对话框、菜单、通知和状态组件 |
| 图标 | @element-plus/icons-vue | 菜单、按钮和状态图标 |
| Markdown | markdown-it | 知识文档预览、上下文包预览和报告展示 |

前端实现原则：

- 所有项目空间页面必须显示当前项目。
- 未选择项目时展示明确空状态。
- 代码分析、向量索引、语义检索、上下文包和影响分析的结果状态应保持解耦。
- 核心可用版只展示基础关系数据，不建设复杂交互式图谱页面。

## 后端服务边界

| 服务 | 职责 |
|---|---|
| AuthService | 登录、当前用户、管理员校验 |
| SystemConfigService | 工作区、状态存储、Qdrant、Neo4j、embedding、对象存储和路径策略配置 |
| ProjectService | 项目、仓库同步、项目成员 |
| CodeAnalysisService | 代码扫描、分析任务、失败文件 |
| CodeFactService | 代码事实查询、接口关系 |
| GraphService | Neo4j 图节点边写入、基础邻居查询、有限深度路径查询和 JSON 图数据导出 |
| KnowledgeDocumentService | 知识文件和模板 |
| KnowledgeAssetService | 知识资产、状态流转、证据绑定 |
| IndexService | 索引构建、索引状态、索引记录 |
| SearchService | 可信检索、结果解释、证据关联 |
| SearchLogService | 检索日志、低命中判定、治理输入和脱敏查询摘要 |
| ContextPackageService | 候选召回、上下文包生成、导出 |
| ChangeAnalysisService | 文件列表和 diff 影响初判 |
| GovernanceService | 治理摘要和治理项 |
| ReviewTaskService | 复审任务创建、处理和忽略 |
| AuditService | 审计记录和查询 |

分析、索引、上下文生成、影响分析都是任务型操作，要有状态、失败原因和可重试入口。普通查询接口不应触发重型任务。

## 状态设计

### 项目状态

- `active`
- `syncing`
- `sync_failed`
- `archived`

### 仓库快照状态

- `created`
- `analyzing`
- `analyzed`
- `partial_failed`
- `failed`

### 知识资产状态

- `draft`
- `confirmed`
- `stale`
- `archived`

### 任务状态

- `queued`
- `running`
- `succeeded`
- `partial_failed`
- `failed`
- `cancelled`

### 索引状态

- `not_built`
- `building`
- `ready`
- `stale`
- `failed`

## 错误处理

| 场景 | 常见错误 | 处理方式 |
|---|---|---|
| 项目接入 | Git 不可访问、分支不存在、鉴权失败、路径非法、项目名重复 | 保留失败原因，允许重试，项目列表显示失败状态 |
| 代码分析 | 路径不存在、编码异常、单文件解析失败、文件过大、不支持语言 | 部分失败不清空成功结果，失败文件可查看 |
| 索引 | embedding 不可用、向量库不可用、metadata 缺失、写入失败 | 最近成功索引继续可用，显示数据新鲜度风险 |
| Neo4j 图写入 | 图数据库不可用、写入失败、节点边缺失 | 保留 MySQL 事实，标记任务部分失败，允许重试图写入 |
| 检索 | 索引未构建、无可检索数据、向量库异常、未授权来源 | 返回明确空状态或权限错误，不能泄露未授权项目摘要 |
| 上下文包 | 召回为空、内容超长、引用失效、导出权限不足 | 允许手动添加上下文项，导出前再次校验权限 |
| 变更影响 | diff 非法、文件列表为空、分析基线缺失、无法定位符号 | 区分无法分析和未发现影响，说明不确定性来源 |

## 非功能要求

### 安全

- 所有项目数据必须按 `projectId` 隔离。
- 所有项目接口必须做后端权限校验。
- 文件路径必须限制在项目工作区或配置的知识库路径内。
- 上下文包导出前必须重新校验权限。
- 上下文包导出前必须执行基础敏感信息过滤。
- 审计不保存密码、token、私钥或完整敏感内容。

### 可靠性

- 耗时操作必须任务化。
- 任务失败不应清空上一份可用结果。
- 任务必须记录失败原因、开始时间、结束时间和触发人。
- 重试任务必须生成新任务记录，不能覆盖历史失败原因。

### 性能

- 核心可用版面向中小型项目。
- 列表接口必须分页。
- 检索必须支持 topK 和来源过滤。
- Neo4j 图查询、图谱和调用关系接口只返回裁剪后的子集。

### 可追溯

- 检索结果必须能追溯到源对象。
- 上下文包必须保存引用清单。
- 知识确认必须保存确认人和确认时间。
- 影响分析必须保存输入原文、解析结果和置信度。

### 可扩展

- 默认向量库为 Qdrant，向量库、embedding、重排模型应通过配置抽象保留替换能力。
- 核心可用版默认图数据库为 Neo4j，图节点边 schema 必须通过 `projectId`、`snapshotId`、`sourceId` 和 `targetId` 与 MySQL/Qdrant 对齐。
- 内置 AI 问答必须复用可信检索和上下文包。
- commit、PR、MR 影响分析必须复用文件列表和 diff 分析能力。
- 跨项目检索必须在项目隔离模型之上扩展。

## 核心验收标准

| 场景 | 验收条件 |
|---|---|
| 项目与权限 | 用户只能看到被授权项目；未授权项目 API 返回 403 |
| 系统配置 | 管理员可以查看、保存并测试 MySQL、Qdrant、Neo4j、embedding、对象存储和路径策略配置 |
| 代码事实 | 系统能产出文件、类、方法、接口、SQL、调用关系等代码事实 |
| Neo4j 图关系 | 系统能把基础代码关系、接口关系、证据关系和知识关系写入 Neo4j |
| 知识资产 | 成员可以创建草稿知识，负责人可以确认、标记待复审、归档知识 |
| 统一索引与检索 | 系统可以构建当前项目索引，检索必须带项目过滤 |
| 检索结果复用 | 后端重新校验权限和源对象有效性 |
| AI 上下文包 | 上下文包保存引用清单，导出记录审计 |
| 变更影响 | 系统输出变更文件、命中代码事实、候选影响、风险和测试建议 |
| ReviewTask | 处理后回写 KnowledgeAsset 或 Evidence 状态，并触发索引和图关系更新 |
| 低命中治理 | 低命中或无结果查询生成脱敏 SearchLog |
| 审计追踪 | 权限变更、知识确认、上下文导出和任务执行都能查询 |
