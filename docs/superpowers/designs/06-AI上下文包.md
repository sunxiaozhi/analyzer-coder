# AI 上下文包详细设计

## 设计职责

AI 研发助手模块负责基于任务描述召回候选上下文，支持用户勾选、排除、排序，生成 Markdown 或纯文本上下文包，保存引用清单，写入 Neo4j 引用关系，并记录导出审计。

核心可用版只生成上下文包，不直接调用大模型写代码。

## 数据对象

### ContextPackage

```text
id
projectId
taskType
taskText
selectedItems
excludedItems
content
format
createdBy
createdAt
exportedAt
```

上下文包必须保存引用清单，不能只保存最终文本。导出必须写审计。

## API 设计

- `POST /api/projects/{projectId}/context-packages/preview`
- `POST /api/projects/{projectId}/context-packages`
- `GET /api/projects/{projectId}/context-packages`
- `GET /api/projects/{projectId}/context-packages/{packageId}`
- `POST /api/projects/{projectId}/context-packages/{packageId}/export`

增强版预留：

- `POST /api/projects/{projectId}/ai/chat`

## 服务边界

| 服务 | 职责 |
|---|---|
| ContextPackageService | 候选召回、上下文包生成、导出 |
| SearchService | 根据任务召回候选上下文 |
| SearchLogService | 记录任务检索和低命中情况 |
| GraphService | 写入 ContextPackage 与引用对象的 INCLUDES 关系 |
| AuditService | 记录上下文包导出 |

## 上下文包数据流

```text
输入任务
  -> 根据任务类型生成召回策略
  -> 调用可信检索生成候选上下文
  -> 记录检索和上下文候选来源
  -> 用户勾选、排除、排序
  -> 生成上下文包正文
  -> 保存 ContextPackage 和引用清单
  -> 写入 Neo4j INCLUDES 关系
  -> 导出前执行权限复检和基础敏感信息过滤
  -> 导出时写入 AuditEvent
```

## 上下文内容结构

上下文包必须包含：

- 任务描述。
- 用户约束。
- 已确认知识。
- 待复审知识风险提示。
- 相关代码事实。
- 相关接口、SQL 和调用链。
- 风险点和测试建议。
- 引用来源清单。
- 生成时间。

## 任务类型召回策略

| 任务类型 | 召回重点 |
|---|---|
| 需求开发 | 业务规则、接口、模块说明、开发规范 |
| 缺陷修复 | 故障案例、相关代码、历史知识、SQL 线索 |
| 接口变更 | 前后端映射、接口说明、测试建议 |
| 重构 | 模块边界、调用关系、ADR、规范 |
| 排障 | 故障案例、接口、SQL、日志线索 |
| 代码评审 | 开发规范、风险点、变更影响、相关知识 |

## 导出规则

- 用户只能引用有权限访问的项目资料。
- 导出前必须重新校验权限。
- 导出前必须执行基础敏感信息过滤。
- 导出必须记录 AuditEvent。
- 导出审计保存引用清单，不保存敏感明文。

## 页面设计

| 页面 | 核心功能 |
|---|---|
| AI 研发助手 | 输入任务，召回候选，上下文勾选排除排序，生成和导出上下文包，展示风险和测试建议 |

## 验收标准

- 用户可以输入任务描述并召回候选上下文。
- 用户可以勾选、排除、排序上下文项。
- 系统生成 Markdown 或纯文本上下文包。
- 上下文包保存引用清单。
- 上下文包生成后，系统写入 ContextPackage 与引用对象的 Neo4j 关系。
- 导出上下文包记录审计。
- 导出前执行权限复检和基础敏感信息过滤。
