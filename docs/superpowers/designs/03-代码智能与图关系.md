# 代码智能与图关系详细设计

## 设计职责

代码智能模块负责仓库代码分析、CodeFact 生成、接口识别、SQL 线索识别、基础调用关系生成、Neo4j 图写入、基础图查询和图数据导出。

## 数据对象

### CodeFact

```text
id
projectId
snapshotId
factType: file | class | method | endpoint | sql | call | module | frontend_call
filePath
symbolName
signature
startLine
endLine
metadata
confidence
```

CodeFact 是检索对象、知识证据、上下文包引用和变更影响匹配目标。

## Neo4j 图模型

### 节点类型

- Project
- RepositorySnapshot
- File
- Class
- Method
- Endpoint
- SQL
- FrontendCall
- KnowledgeAsset
- Evidence
- IndexRecord
- ContextPackage
- ChangeTask
- ReviewTask

### 边类型

| 边类型 | 起点 -> 终点 | 说明 |
|---|---|---|
| HAS_SNAPSHOT | Project -> RepositorySnapshot | 项目拥有仓库快照 |
| CONTAINS | Project -> File | 项目包含文件 |
| DECLARES | File -> Class/Method/Endpoint/SQL | 文件声明代码事实 |
| HAS_METHOD | Class -> Method | 类型包含方法 |
| EXPOSES | Method -> Endpoint | 方法暴露 HTTP 接口 |
| CALLS | Method -> Method | 方法调用方法 |
| USES_SQL | Method -> SQL | 方法使用 SQL 线索 |
| FRONTEND_CALLS | FrontendCall -> Endpoint | 前端调用后端接口 |
| EVIDENCES | Evidence -> CodeFact | 证据指向代码事实 |
| SUPPORTS | Evidence -> KnowledgeAsset | 证据支撑知识资产 |
| INDEXES | IndexRecord -> CodeFact/KnowledgeAsset/File | 索引记录来源对象 |
| RETRIEVES | SearchResult -> IndexRecord | 检索结果命中索引记录 |
| INCLUDES | ContextPackage -> KnowledgeAsset/CodeFact | 上下文包引用知识或代码 |
| IMPACTS | ChangeTask -> CodeFact/Endpoint/KnowledgeAsset | 变更任务影响对象 |
| REQUIRES_REVIEW | ChangeTask -> ReviewTask | 变更触发复审候选 |

## API 设计

- `POST /api/projects/{projectId}/analysis`
- `GET /api/projects/{projectId}/analysis/tasks`
- `GET /api/projects/{projectId}/analysis/tasks/{taskId}`
- `GET /api/projects/{projectId}/code-facts`
- `GET /api/projects/{projectId}/code-facts/{factId}`
- `GET /api/projects/{projectId}/code-graph`
- `GET /api/projects/{projectId}/code-graph/neighbors`
- `GET /api/projects/{projectId}/code-graph/paths`
- `GET /api/projects/{projectId}/interface-map`

`code-graph` 系列接口由 Neo4j 支撑。核心可用版只要求项目内节点邻居查询、有限深度路径查询和裁剪后的 JSON 节点边导出，所有查询必须带 `projectId` 过滤，并限制最大返回节点数和路径深度。

## 服务边界

| 服务 | 职责 |
|---|---|
| CodeAnalysisService | 代码扫描、分析任务、失败文件 |
| CodeFactService | 代码事实查询、接口关系 |
| GraphService | Neo4j 图节点边写入、基础邻居查询、有限深度路径查询和 JSON 图数据导出 |

## 代码事实生成数据流

```text
触发分析
  -> 创建分析任务
  -> 扫描文件
  -> 解析 Java/Spring/MyBatis/前端 API
  -> 生成 CodeFact
  -> 生成接口关系和调用关系
  -> 写入 Neo4j 图节点边
  -> 保存成功事实和失败文件
  -> 更新 RepositorySnapshot
  -> 写入 AuditEvent
```

单文件失败不导致整个任务失败。任务整体失败时保留上一版可用分析结果。Neo4j 写入失败时，分析任务应标记为 `partial_failed` 或 `failed`，并保留 MySQL 中已保存的 CodeFact 和失败原因，便于用户重试图数据写入。

## 证据有效性检查

代码分析完成后必须执行证据有效性检查：

```text
当前 RepositorySnapshot
  -> 对比已绑定 Evidence 的文件路径、符号、行区间和签名
  -> 无法定位 Evidence
  -> 生成治理项或 ReviewTask
  -> 不直接改变 KnowledgeAsset 状态
```

## 图规则

- 所有节点和边必须包含 `projectId`。
- 跨快照关系必须包含 `snapshotId` 或 commit 范围。
- `CALLS`、`IMPACTS`、`REQUIRES_REVIEW` 等推断边必须包含 `confidence` 和 `reason`。
- Qdrant 中的向量数据用于语义召回，Neo4j 中的图数据用于关系追踪，两者通过 `sourceId`、`targetId`、`projectId` 和 `snapshotId` 对齐。
- 图查询只返回裁剪后的子集。

## 页面设计

| 页面 | 核心功能 |
|---|---|
| 代码事实 | 触发分析，查看代码事实列表和详情，查看失败文件，导出基础图数据，绑定证据或加入上下文 |
| 接口关系 | 查看 endpoint、前端调用、匹配关系和接口风险；接口说明通过 `api_doc` 知识资产承载 |

## 验收标准

- 用户可以同步仓库并触发代码分析。
- 系统能产出文件、类、方法、接口、SQL、调用关系等代码事实。
- 系统能把基础代码关系、接口关系、证据关系和知识关系写入 Neo4j。
- 单文件失败时任务为部分失败，成功事实仍可用。
- 分析结果绑定到明确仓库快照。
- 系统可以从 Neo4j 查询并导出裁剪后的基础 JSON 图数据。
