# 变更影响分析详细设计

## 设计职责

变更影响分析模块负责接收文件列表或 diff，匹配直接变更 CodeFact，使用 Neo4j 展开有限深度关系路径，使用 Qdrant 补充语义候选，输出候选影响、风险、测试建议和待复审候选。

影响分析定位为初判，不输出绝对结论。

## 数据对象

### ChangeAnalysisTask

```text
id
projectId
snapshotId
inputType: file_list | diff
inputContent
changedFiles
matchedCodeFacts
impactedEndpoints
impactedKnowledgeAssets
risks
testSuggestions
confidence
status
createdBy
createdAt
```

影响结果必须区分事实和候选推断。标记知识待复审必须由项目负责人确认。

## API 设计

- `POST /api/projects/{projectId}/change-analysis`
- `GET /api/projects/{projectId}/change-analysis`
- `GET /api/projects/{projectId}/change-analysis/{taskId}`
- `POST /api/projects/{projectId}/change-analysis/{taskId}/review-candidates/{assetId}/confirm`

## 服务边界

| 服务 | 职责 |
|---|---|
| ChangeAnalysisService | 文件列表和 diff 影响初判 |
| CodeFactService | 匹配直接变更代码事实 |
| GraphService | 展开 Neo4j 有限深度影响路径，写入 IMPACTS/REQUIRES_REVIEW |
| SearchService | 通过 Qdrant 补充语义相关候选 |
| ReviewTaskService | 创建待复审候选或 ReviewTask |
| AuditService | 记录影响分析任务 |

## 变更影响数据流

```text
输入文件列表或 diff
  -> 创建 ChangeAnalysisTask
  -> 解析变更文件和行区间
  -> 匹配 CodeFact
  -> 使用 Neo4j 展开有限深度影响路径
  -> 使用 Qdrant 补充语义相关候选
  -> 查找 Evidence
  -> 找到 KnowledgeAsset
  -> 输出影响接口、影响知识、风险和测试建议
  -> 生成待复审候选或 ReviewTask
  -> 写入 Neo4j IMPACTS/REQUIRES_REVIEW 关系
  -> 写入 AuditEvent
```

## 结果分层

- 确定事实：变更文件、变更行、直接命中的代码事实。
- 高置信候选：直接命中的 endpoint、SQL、绑定知识。
- 中置信候选：Neo4j 有限深度路径内的调用关系、接口和模块影响。
- 低置信候选：模块级、目录级、关键词或 Qdrant 语义召回影响。

## 置信度规则

- 直接匹配 CodeFact 为确定事实。
- 直接绑定 Evidence 的知识为高置信候选。
- Neo4j 一跳内影响为高或中置信候选，取决于关系类型和置信度。
- Neo4j 多跳、模块级和目录级影响为中低置信候选。
- Qdrant 语义召回仅作为补充候选，默认低于直接图关系。

## 边界

核心可用版不要求：

- commit 分析。
- 分支对比。
- PR/MR 集成。
- CI 自动触发。
- 完整跨语言调用图。
- 复杂运行时依赖推断。
- 自动生成最终测试计划。
- 自动判断业务规则冲突。

commit、分支对比和 PR/MR 范围属于增强版输入。

## 错误处理

| 场景 | 处理方式 |
|---|---|
| diff 非法 | 返回明确错误，不创建成功任务 |
| 文件列表为空 | 返回输入校验错误 |
| 分析基线缺失 | 提示先执行代码分析 |
| 无法定位符号 | 区分无法分析和未发现影响 |
| 调用链不完整 | 标注不确定性来源 |

|---|---|
| 变更影响 | 输入文件列表或 diff，查看候选影响、风险、测试建议和待复审候选 |

## 验收标准

- 用户可以输入文件列表或 diff。
- 系统输出变更文件、命中代码事实、候选接口影响、候选知识影响、风险和测试建议。
- 结果标注置信度和不确定性。
- 影响分析使用 MySQL 命中直接 CodeFact，使用 Neo4j 展开有限深度关系路径，并可使用 Qdrant 补充语义候选。
- 命中已确认知识时生成待复审候选或 ReviewTask。
- 最终标记待复审必须由项目负责人确认。
