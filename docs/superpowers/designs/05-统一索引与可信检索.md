# 统一索引与可信检索详细设计

## 设计职责

统一索引与检索模块负责构建 Qdrant 语义索引、管理 IndexRecord、执行项目内可信检索、生成 SearchResult、记录 SearchLog，并支持检索结果复用为上下文项、知识资产草稿或 Evidence。

## 数据对象

### IndexRecord

```text
id
projectId
snapshotId
sourceType: code | knowledge_doc | knowledge_asset | endpoint | sql | call_chain
sourceId
chunkId
title
text
vectorId
metadata
status
indexedAt
```

核心可用版默认使用 Qdrant 保存向量数据，`vectorId` 对应 Qdrant point id。所有检索必须带 `projectId` 过滤。索引记录必须能追溯回源对象。

### SearchResult

```text
projectId
resultSnapshotId
sourceType
sourceId
chunkId
title
snippet
score
status
updatedAt
evidenceCount
explanation
```

SearchResult 可以加入上下文包，也可以绑定为知识证据。复用检索结果时，后端必须使用 `sourceType`、`sourceId`、`chunkId` 或 `resultSnapshotId` 重新校验项目权限和源对象有效性，不能只信任前端传回的文本片段。

### SearchLog

```text
id
projectId
queryHash
querySummary
filters
topScore
resultCount
isLowHit
usedForContext
createdBy
createdAt
```

SearchLog 必须脱敏保存。低命中规则由 `resultCount`、`topScore` 和用户反馈共同决定，核心可用版可以先采用可配置阈值。

## API 设计

### 统一索引

- `POST /api/projects/{projectId}/index/build`
- `GET /api/projects/{projectId}/index/status`
- `GET /api/projects/{projectId}/index/records`

### 可信检索

- `POST /api/projects/{projectId}/search`
- `POST /api/projects/{projectId}/search/results/{resultId}/copy`
- `GET /api/projects/{projectId}/search/logs`

检索结果必须包含 `sourceType`、`sourceId`、`title`、`snippet`、`score`、`status`、`updatedAt`、`evidenceCount` 和 `explanation`。

`resultId` 对应后端短期保存的 SearchResultSnapshot。若未保存快照，复用类接口必须接收 `sourceType`、`sourceId` 和 `chunkId`，由后端重新读取源对象并校验权限。SearchLog 用于治理低命中查询，仅项目负责人和管理员可查看明细。

## 服务边界

| 服务 | 职责 |
|---|---|
| IndexService | 索引构建、索引状态、索引记录 |
| SearchService | 可信检索、结果解释、证据关联 |
| SearchLogService | 检索日志、低命中判定、治理输入和脱敏查询摘要 |
| GraphService | IndexRecord 与源对象关系同步 |

## 统一索引数据流

```text
触发索引
  -> 创建索引任务
  -> 读取 CodeFact、知识文档、知识资产、接口、SQL 和调用链
  -> 生成统一 chunk
  -> 写入 IndexRecord
  -> 写入默认 Qdrant 向量库
  -> 更新索引状态
  -> 写入 AuditEvent
```

新索引构建成功前不覆盖旧索引。embedding 或 Qdrant 失败时，最近成功索引继续可用。

## 可信检索数据流

```text
输入问题
  -> 校验项目权限
  -> 强制带 projectId 过滤
  -> 查询索引
  -> 读取 metadata 和源对象
  -> 关联知识状态、证据和更新时间
  -> 记录 SearchLog 和可选 SearchResultSnapshot
  -> 返回可追溯 SearchResult
```

不允许只返回文本片段。每条结果必须能跳转到源对象。

## 低命中规则

- `resultCount = 0` 直接记为低命中。
- `topScore` 低于项目配置阈值时记为低命中。
- 用户手动反馈“不相关”时记为低命中。
- SearchLog 只保存脱敏 query 摘要和过滤条件，不保存敏感原文。

## 状态设计

索引状态：

- `not_built`
- `building`
- `ready`
- `stale`
- `failed`

## 页面设计

| 页面 | 核心功能 |
|---|---|
| 统一索引 | 构建索引，查看索引状态、来源分布、记录和失败原因 |
| 可信检索 | 自然语言检索，过滤来源，查看证据，复制结果，加入上下文包，绑定为证据，生成知识资产草稿 |

## 验收标准

- 系统可以构建当前项目索引。
- 检索必须带项目过滤。
- 检索结果展示来源、标题、分数、状态、证据数量和推荐原因。
- 已确认知识优先展示，待复审知识提示风险。
- 检索结果可以复制、加入上下文包、绑定为证据或生成知识资产草稿。
- 检索结果复用时，后端使用 `sourceType`、`sourceId`、`chunkId` 或 `resultSnapshotId` 重新校验权限和源对象有效性。
- 低命中、无结果或用户反馈不相关的查询会生成脱敏 SearchLog。
- 索引记录写入 Qdrant 时 payload 包含 `projectId`、`sourceType`、`sourceId`、`chunkId`、`status`、`filePath`、`title` 和 `updatedAt`。
